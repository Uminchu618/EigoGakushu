# 宿題プリント暗記アプリ 企画メモ（初期案）

## 1. 目的
- 学校の不規則動詞プリント（原形・過去形・過去分詞・意味）を**短時間で覚える**。
- 「書いて覚える」前に、アプリで「出る順に思い出す練習」をして定着率を上げる。

## 2. プリント分析（画像から）
- 内容は不規則動詞の活用（例: cut/cut/cut, come/came/come など）。
- 問題形式は「意味→活用」「原形→過去形」「原形→過去分詞」など、複数方向の想起が必要。
- 同じ語を反復して書く余白があり、反復練習が前提。

## 3. アプリのコンセプト

### コンセプト名
**3方向クイズ + 忘却対策リピート**

### コア体験
1. 1問ずつ出題（例: 「来る」→原形は？過去形は？過去分詞は？）
2. 自分で入力
3. 即時採点（スペル一致）
4. 間違えた問題は短い間隔で再出題
5. 最後に「間違えた単語だけ」再テスト

## 4. 最初に作るべき最小機能（MVP）

### 必須
- 単語データ管理（原形・過去形・過去分詞・意味）
- 出題モード
  - 意味→3活用
  - 原形→過去形
  - 原形→過去分詞
- 入力採点（大文字小文字は区別しない）
- 苦手単語の再出題キュー
- 結果画面（正答率・苦手一覧）

### あると効果が高い
- 音声読み上げ（発音とセットで記憶）
- タイマー（1セット5分）
- 学習履歴（前日比）

## 5. 学習効率を上げる設計ルール
- **想起ベース**: 先に答えを見せない。
- **間違い優先**: 間違えた単語ほど多く出す。
- **短時間反復**: 1セッション5〜10分。
- **多方向出題**: 意味↔英語、原形→過去形/過去分詞を混ぜる。

## 6. 画面イメージ
1. ホーム
   - 「今日の5分を始める」ボタン
   - 前回正答率
2. クイズ画面
   - 問題文
   - 入力欄
   - 判定（○/×）
3. 結果画面
   - スコア
   - 要復習リスト
   - 「間違いだけ再挑戦」

## 7. 推奨技術（最初の一歩）
- まずはスマホで使いやすい**Webアプリ**（PWA）
- フロント: React / Next.js
- 保存: LocalStorage（最初はサーバー不要）

## 8. 次の具体化タスク
1. 対象単語（プリント範囲）をCSV化（`verbs.csv`）
2. 出題ルール（何問で1セットか）を固定
3. 画面ワイヤーを3画面だけ作成
4. MVPを実装
5. 1週間使って改善
